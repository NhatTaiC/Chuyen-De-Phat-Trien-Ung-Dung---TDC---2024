1. Feature Function
+ Layout (Users/Admin)
+ Feature Name -> UI (Users/Admin) -> Screen Description -> Workflow: table workflow/activity

2. Non-feature Function:
+ Enviroment, Capacity, Multitasking, Performance, Safe, Security,....

III. Use-case Diagram
1. Actors: Admin, User, Database
2. Use-cases: CRUD member, Cart, CRUD stores,.. CRUD products

	3.2. Use-cases Owner Name
	3.3. Brief Description (mo ta kich ban ngan gon: admin lam duoc gi? user lam duoc gi?)
	3.4. Flow of Event: Basic flow/Alternative flow
	3.5. Special requirements
	3.6. Pre-conditions
	3.7. Post-conditions
	3.9. Extension points: Searching, Printing,...

IV. Data tables
+ Design Data Tables: Data Types, Primary Keys, Foreign Keys,....
